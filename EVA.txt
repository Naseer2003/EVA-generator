# 4. Why Python Should Handle EVA

Do NOT implement heavy statistics in Node.js.

Use Python because:

* SciPy EVT libraries exist
* Optimization is stable
* Numerical precision is better
* Scientific ecosystem is mature

Node.js should:

* orchestrate
* authenticate
* manage workflows
* serve APIs

Python should:

* run EVA calculations
* perform MLE
* run optimization
* generate statistical outputs

# 8. Statistical Engine Design

This is the most important part.

---

# Step 1 — Data Validation

Validate:

* missing values
* duplicate timestamps
* unit consistency
* impossible values

---

# Step 2 — Peak Extraction

Support:

* Annual maxima
* POT (Peak Over Threshold)

Future-ready design.

---

# Step 3 — Distribution Selection

Initially:

* Gumbel

Later add:

* GEV
* Weibull
* Fréchet
* Generalized Pareto

---

# Step 4 — Parameter Estimation

Use:

* MLE (primary)
* MoM (initialization)

Optimization:

* L-BFGS-B

---

# Step 5 — Numerical Stabilization

CRITICAL.

Use:

```python id="b65k88"
logsumexp
np.clip
epsilon safeguards
parameter bounds
```

Without this:

* your optimizer will explode.

---

# Step 6 — Goodness-of-Fit

Implement:

* Anderson-Darling
* KS Test
* QQ plots
* PP plots

---

# Step 7 — Confidence Intervals

Production-grade approach:

1. Fisher Information Matrix
2. Bootstrap resampling
3. Profile likelihood

---

# Step 8 — Return Level Computation

Support:

* 2-year
* 5-year
* 10-year
* 25-year
* 50-year
* 100-year

Use:

x_T = \mu - \beta \ln\left[-\ln\left(1-\frac1T\right)\right]
